\datedsubsection{\textbf{CompIntel Research | 基于 LangGraph 的 Multi-Agent 竞品情报分析系统}}{}
\textbf{技术栈：} Python · LangGraph · Qdrant (BM25+Dense+RRF) · FastAPI + WebSocket · Pydantic v2 · DeepSeek V4

\textbf{项目概述：}输入公司名全自动产出竞品画像、市场格局、SWOT 与结构化报告。10 节点 StateGraph 编排，扇出并行采集 + 证据分级 + 编辑统稿 + LLM-as-Judge 门控 + 报告通过后写回 RAG 形成自增长记忆。核心探索有状态多 Agent 编排、LLM 不可靠时的系统鲁棒性、以及离线评估量化 Agent 输出质量。

\begin{itemize}
  \item \textbf{有状态多 Agent 编排：}\texttt{StateGraph} 10 节点主图 + 3 节点并行子图（Search / Scrape / RAG）。\texttt{Send} 按竞品数扇出 fork 独立子图三路并行，Reducer 自动归并，SqliteSaver Checkpointer 支持断点续跑及 thread\_id 隔离。子图任一 worker 失败不丢其他 worker 已采集数据，5 竞品并行加速比约 3$\times$。
  \item \textbf{LLM 可靠性体系：}全 Agent 三层降级（LLM → Derived → Template），LLM API 不可用全链路不中断。双阶段推理管道（Reasoner → Formatting），Compression Bridge 将 4000+ token chain-of-thought 本地压缩至 ≤400 字符，避免格式化模型自发推理。调优后 LLM 调用失败率 30-40\% → <5\%，单次查询调用数 17 → <10，wall-clock 22min → <5min（3 竞品）。确定性缓存消除 Reviewer 修订回环中的重复调用。
  \item \textbf{证据分级 + 编辑统稿 + 质量门控：}Curator 按数据密度分四级（rich/adequate/thin/empty），含 404/验证码/社交链接等垃圾过滤规则，empty 级禁止编造 SWOT。Editor 做术语统一、去重、矛盾检测。Reviewer 三维加权门控（0.4/0.4/0.2，≥7 放行），3 次退回重写不过自动放行 + 人工审阅标注。
  \item \textbf{RAG 自增长记忆：}Qdrant BM25 稀疏 + Dense（BGE-small-zh 512 维）+ RRF 混合检索，\texttt{:memory:} 模式零 Docker。报告审核通过后自动写回向量库，形成读写闭环。Embedding 协议抽象双实现，\texttt{Semaphore(3)} 全局 LLM 并发控速。
  \item \textbf{离线质量评估：}自建 7 维评估框架（0-14 分），纯规则从 snapshot.json + report.md 直接打分（竞品真实性/赛道准确性/RAG 记忆/Source 追溯/降级行为/Pipeline 完整性/报告可读性）。与 Reviewer 互补——前者看事实质量，后者看结构质量。45 测试覆盖全链路 + 降级双路径。
\end{itemize}
